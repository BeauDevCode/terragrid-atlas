# Security Policy

## Reporting

If you find a security issue, do not publish private details in a public issue.

Send a private report to the maintainer with:

- a short summary
- steps to reproduce
- affected files or pages
- suggested fix, if known

## Sensitive data

Do not commit:

- API keys
- `.env` files
- private datasets
- private logs
- screenshots with private information
- tokens or deployment secrets

## Project safety notes

- Sample data should stay clearly labeled.
- Private data should not be added to the public repo.
- Environment variables should be used for secrets.
- Contributors should avoid adding private or restricted datasets.

## Supported scope

This is a portfolio web app. Security reports should focus on the public repository, local setup, sample data, configuration, and demo behavior.
