## Full Description

I built TerraGrid Atlas to make global energy infrastructure more visible and easier to explore. While learning about power systems, I kept running into the same problem: useful infrastructure data exists, but it is often fragmented across different sources, hard to visualize, incomplete, or locked behind specialized tools.

TerraGrid Atlas brings the core pieces into one map-first web app. It combines a global infrastructure map, power plants, transmission systems, substations, data centers, country dashboards, comparison tools, generation mix charts, price signals, load snapshots, grid regions, alerts, and a transparent methodology page into a single research-friendly interface.

The project is built with Next.js, TypeScript, Tailwind CSS, MapLibre GL JS, Recharts, and a typed local data architecture. The current MVP uses transparent sample/demo and estimated data so it can run without paid services or a required database. It is not presented as fully live global infrastructure coverage. The code is structured around provider adapters so real public data sources such as OpenStreetMap/Overpass, public power plant datasets, regional load feeds, country energy statistics, and outage/event feeds can be connected later.

My goal was to create something that feels like a serious infrastructure intelligence platform, not just a school project. I wanted students, researchers, and curious citizens to be able to open one tool and start understanding how power systems are distributed and connected around the world.

## Tools Used

Next.js, TypeScript, Tailwind CSS, MapLibre GL JS, Recharts, lucide-react, GitHub, Vercel, and Canva-ready showcase assets.

## What Makes It Original

TerraGrid Atlas is not a chatbot. It combines geospatial mapping, infrastructure intelligence, and dashboard analytics into one visual research tool for exploring how energy systems are distributed globally.
